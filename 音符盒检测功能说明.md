# SongPlayer 音符盒检测功能改进

## 概述

我们成功为SongPlayer模组添加了两种音符盒检测方式，允许用户根据需要选择不同的检测模式。

## 功能特性

### 1. 检测模式

#### NBT_DATA 模式（默认）
- **原理**: 通过检测音符盒的NBT数据来判断音色和音调
- **优点**: 精确，能获取完整的音符盒状态信息
- **适用场景**: 当音符盒已经正确配置NBT数据时

#### BLOCK_BASED 模式（新增）
- **原理**: 通过检测音符盒下方的方块类型来判断音色
- **优点**: 更直观，符合Minecraft原版音符盒机制
- **适用场景**: 当需要根据实际方块布局来确定音色时

## 使用方法

### 命令

#### 查看当前检测模式
```
$setNoteblockDetectionMode
```
或使用别名：
```
$noteblockMode
$detectionMode
```

#### 设置检测模式为NBT数据检测
```
$setNoteblockDetectionMode NBT_DATA
```

#### 设置检测模式为方块检测
```
$setNoteblockDetectionMode BLOCK_BASED
```

### 配置文件

检测模式设置会自动保存到配置文件中：
```json
{
  "noteblockDetectionMode": "NBT_DATA"
}
```

## 实现细节

### 新增文件

1. **NoteblockDetectionMode.java**
   - 定义检测模式枚举
   - 包含显示名称

2. **BlockBasedInstrumentDetector.java**
   - 实现基于方块的音色检测逻辑
   - 支持多种方块类型识别

### 修改文件

1. **Config.java**
   - 添加检测模式配置选项
   - 默认值为NBT_DATA模式

2. **Stage.java**
   - 修改`checkBuildStatus`方法支持两种检测方式
   - 添加`detectNoteblocksByBlock`方法
   - 添加`getNoteblockPitch`方法

3. **CommandProcessor.java**
   - 添加`setNoteblockDetectionModeCommand`命令
   - 支持命令自动补全

## 技术特点

### 兼容性
- 保持与原有NBT检测方式的完全兼容
- 新增的方块检测作为可选功能
- 配置可在运行时切换

### 扩展性
- 方块类型映射易于扩展
- 检测逻辑模块化设计
- 支持添加新的检测模式

### 用户体验
- 中文界面友好
- 命令支持别名
- 实时配置保存

## 注意事项

1. **方块检测模式下的音调**：
   - 默认音调为0
   - 玩家需要手动右键点击音符盒来调整音调

2. **性能考虑**：
   - 方块检测需要查询世界方块状态
   - 在大型舞台上可能有轻微性能影响

3. **兼容性**：
   - 当前实现针对Minecraft 1.21
   - 部分API可能需要根据版本调整

## 总结

这个改进为SongPlayer提供了更灵活的音符盒检测方式，用户可以根据实际需求选择最适合的检测模式。基于方块的检测方式更符合Minecraft原版机制，而NBT检测方式则提供了更精确的控制。
